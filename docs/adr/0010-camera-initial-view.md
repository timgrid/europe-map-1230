# ADR-0010: Динамический fit-to-view камеры по размеру карты

Дата: 2026-06-04

## Статус

**Accepted**

## Контекст

До этого изменения камера была жёстко зафиксирована в `App.tsx`:
`camera={{ position: [cx, 180, -cy + 25], fov: 18 }}`. Расчёт видимой области при
fov=18° и дистанции ~182 показывал, что пользователь видит только ~57 единиц ширины
на ground plane — это **~19% от ширины карты 300 единиц**. Остальная часть Европы
(включая Скандинавию, Средиземноморье, Восточную Европу) обрезалась за пределами
экрана. OrbitControls давали `maxDistance: 400`, чего было недостаточно для зума-аута
на полную карту.

Дополнительно размер карты по высоте варьируется по годам: от 115 (1400) до 279 (1530)
— то есть оптимальная дистанция камеры должна зависеть от года.

Аналогично aspect ratio экрана (мобильный портрет vs десктоп landscape) меняет
необходимую дистанцию: на узком экране нужно уйти дальше, чтобы уместить ширину.

## Решение

1. **Утилита `getInitialCameraConfig()`** в `src/utils/camera.ts` считает
   дистанцию камеры по формуле:
   ```
   dForWidth  = (mapWidth / 2)  / tan(hFov/2) / cos(polarAngle) * padding
   dForHeight = (mapHeight / 2) / tan(vFov/2) / cos(polarAngle) * padding
   distance   = clamp(max(dForWidth, dForHeight), minDistance, maxDistance)
   ```
   Используется **вертикальный fov** (как в three.js); горизонтальный выводится
   через `2 * atan(tan(vFov/2) * aspect)`. Деление на `cos(polarAngle)` компенсирует
   наклон камеры (ground plane наклонён относительно view direction).

2. **`CameraRig` компонент** (`src/components/CameraRig.tsx`) императивно обновляет
   позицию камеры и fov при изменении `position`/`target`/`fov` props. Это
   необходимо, потому что prop `camera` на `<Canvas>` применяется только на mount.

3. **Параметры** (вынесены константами в App.tsx):
   - `CAMERA_FOV = 20` (было 18; +11% покрытия при той же дистанции, укладывается
     в ограничение AGENTS.md «fov ≤ 30»);
   - `CAMERA_POLAR = 0.2` рад (~11.5° от вертикали — небольшой наклон для 2.5D);
   - `CAMERA_PADDING = 1.15` (15% запаса по краям);
   - `CAMERA_MIN_DIST = 200`, `CAMERA_MAX_DIST = 1500`;
   - OrbitControls `min/maxDistance` привязаны к этим константам.

4. **Resize listener** в App.tsx обновляет `aspect` state при `resize` и
   `orientationchange`, что вызывает пересчёт `cameraFit` и перемещение камеры.

5. **Fog и shadow camera bounds** расширены:
   - `<fog args={['#0a1628', 500, 1200]}>` (было 300/600);
   - shadow camera `left/right/top/bottom = ±180` (было ±150);
   - Canvas `far = 2000` (было 800) — на дальних орбитах камера не упирается в far plane.

6. **Высота карты 222 (1200) → дистанция ~740** на 16:9 десктопе; **279 (1530) → ~940**.
   На мобильном портрете (aspect 9:16) утилита клампит дистанцию в `maxDistance=1500`
   — карта обрезается по бокам, пользователь может поворачивать/панорамировать.

## Последствия

**Плюсы:**
- При загрузке видна вся карта — нет «слепых зон» по краям;
- Камера автоматически подстраивается под год (1530 с большей высотой отъезжает дальше);
- Утилита `getInitialCameraConfig` — чистая функция, легко тестируется (15 unit-тестов
  в `tests/camera.test.ts`);
- Resize и orientation change корректно пересчитывают fit.

**Минусы / компромиссы:**
- fov 20 даёт чуть более выраженную перспективу на краях (vs 18) — на длинных
  фокусных расстояниях разница незаметна, но это отход от «идеала fov 18» из AGENTS.md.
  В пределах 20% расширения — допустимо.
- На мобильном портрете (9:16) карта не умещается целиком без зума — пользователю
  нужно поворачивать. Это естественно для картографических приложений (Google Maps,
  Apple Maps ведут себя так же).
- `CameraRig` — дополнительный useEffect с императивной мутацией камеры; конфликт
  с OrbitControls маловероятен (OrbitControls читает camera.position каждый кадр
  и подхватывает новое значение).

**Риски:**
- При экстремально маленьком окне (height < 300px) дистанция может уйти в
  `maxDistance=1500` — карта будет крошечной. Решение: уменьшить padding или
  поднять maxDistance. На текущих устройствах (Telegram Mini App минимум 360×480)
  проблема не возникает.
- При resize во время drag пользователь может потерять контекст. Решение: можно
  отключить auto-reset камеры во время активного drag (будущее улучшение).
