# ADR-0006: Минификация JSON и округление координат в обработанных данных

Дата: 2026-06-02

## Статус

**Accepted**

## Контекст

После ADR-0005 обработанные JSON-файлы занимали 5.4 МБ суммарно для 7 годов, и страница
«висела» на «Loading map» при первой загрузке (особенно на мобильном). Каждое число
координат имело 16 десятичных знаков (`1.234567890123456e+1`), что:
- раздувает размер в 5-7 раз;
- замедляет `JSON.parse` (больше токенов);
- абсолютно бессмысленно при визуализации на масштабе континента (1 см = десятки км).

## Решение

В `process-geojson.cjs`:
1. округлять координаты до **2 десятичных знаков** (`val.toFixed(2)`) — это ~1.1 км
   точности, что на порядок меньше видимого размера страны;
2. писать JSON **без отступов** (без `JSON.stringify(data, null, 2)`).

Дополнительно добавить:
- `.catch()` + `console.error` в `App.tsx` useEffect;
- `.finally(() => setLoading(false))` чтобы спиннер всегда снимался;
- `LoadingScreen` показывает кнопку «Повторить» через 15 секунд (если CDN медленный).

## Последствия

**Плюсы:** размер с 5.4 МБ → 858 КБ (84% reduction), первая загрузка <1 сек на
десктопе. Лучшая UX при медленном соединении. **Минусы:** округлённые координаты означают,
что два полигона с общей границей могут иметь крошечные (1-2 пикселя) зазоры на границе —
визуально не заметно. **Риски:** если потребуется точность для научного анализа (площадь
территорий), нужно вернуть 6-8 знаков или сохранять оригинал отдельно.
