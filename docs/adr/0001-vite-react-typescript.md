# ADR-0001: Vite + React + TypeScript как основной стек

Дата: 2025-01-01

## Статус

**Accepted**

## Контекст

Проект — интерактивная 3D-карта Европы для веба. Нужно выбрать стек, который:

- даёт быструю разработку с HMR (много итераций над 3D-сценой);
- поддерживает TypeScript из коробки (типобезопасность критична для геометрии);
- легко деплоится на GitHub Pages;
- не тащит тяжёлые зависимости вроде Webpack/Next.

## Решение

Использовать **Vite** (сборщик), **React 18** (UI), **TypeScript** (типизация) и
**Tailwind CSS** (утилитарные стили для UI-оверлея). Vite даёт моментальный HMR и нативный
деплой как статический бандл на GitHub Pages. React Three Fiber поверх three.js — для 3D.

## Последствия

**Плюсы:** быстрый dev-сервер, простой деплой, типобезопасность, зрелая экосистема.
**Минусы:** Vite-специфичные `import.meta.env.BASE_URL` нужны для корректных путей на
GitHub Pages (учтено в dataLoader); `tsc -b` в build-скрипте добавляет ~5-10с к CI.
**Риски:** переход на другой бандлер (Webpack/Rspack) потребовал бы переписать только
`vite.config.ts` и `deploy.yml` — приложение не зависит от Vite API напрямую.
