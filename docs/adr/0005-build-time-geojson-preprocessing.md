# ADR-0005: Препроцессинг GeoJSON на build-time в Node-скрипте

Дата: 2025-01-01

## Статус

**Accepted**

## Контекст

Оригинальные `world_{year}.geojson` файлы:

- содержат весь мир (~10-20 МБ на год, не нужные территории);
- имеют строковые координаты в некоторых полях;
- требуют проекции (d3) и центрирования для 3D-сцены;
- требуют нормализации имён (алиасы, объединения русских княжеств);
- требуют явных цветов (без автоматической палитры).

Если делать это в браузере при загрузке, страница будет долго парсить JSON на каждой
перезагрузке.

## Решение

Написать Node-скрипт **`scripts/process-geojson.cjs`**, который:

1. читает `public/world_{year}.geojson`;
2. фильтрует по bbox Европы;
3. проецирует через d3-geo, центрирует (offset/scale), считает bbox и центр каждой страны;
4. объединяет/переименовывает по конфигу `configs[year]` (например, все русские княжества
   в `Kievan Rus` для 1200 года);
5. назначает цвет из `colorMap` (иначе — fallback на серый `#888`);
6. округляет координаты до 2 знаков и пишет **минифицированный** JSON в
   `public/data/processed/europe_{year}.json`.

Скрипт запускается вручную при добавлении нового года: `node scripts/process-geojson.cjs 1500`.

## Последствия

**Плюсы:** браузер сразу получает готовые данные без парсинга, проекции и нормализации —
только `JSON.parse` и `THREE.Shape` конструктор. Размер уменьшился с 5.4 МБ до 858 КБ
суммарно (84% reduction). **Минусы:** источник `world_{year}.geojson` теперь не в git
(добавлен в .gitignore, скачивается руками), что снижает воспроизводимость. **Риски:** при
смене алгоритма проекции/цветов нужно перегенерировать все года — автоматизировано скриптом.
