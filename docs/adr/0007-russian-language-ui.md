# ADR-0007: Русский язык интерфейса и метаданных

Дата: 2026-06-02

## Статус

**Accepted**

## Контекст

Целевая аудитория — русскоязычные любители истории. Английский интерфейс создавал бы
когнитивный барьер (названия «Kingdom of Castile and León» vs «Кастилия и Леон»).
Технически имена в GeoJSON английские, но описания в `countryMetadata.ts` —
полностью на русском.

## Решение

- Все UI-подписи, тултипы, панели — на русском;
- `countryMetadata.ts` — словарь `id → { name, description }` на русском;
- ID стран в коде — английские slug (`england`, `ottoman_empire`) — для стабильности
  при смене перевода и для консистентности с GeoJSON;
- названия в GeoJSON (`country.name` после `commonAliases`) — на русском, чтобы попадать
  в тултипы без дополнительной транслитерации;
- `idOverrides` и `commonAliases` нормализуют английские варианты написания
  (`Castilla`, `Castille` → `Castile`).

## Последствия

**Плюсы:** нативный опыт для русскоязычной аудитории, перевод можно добавить позже
через копию словаря. **Минусы:** международная аудитория пока не охвачена; код содержит
кириллицу (важно для правильной кодировки файлов — UTF-8). **Риски:** если проект
выйдет на англоязычную аудиторию, потребуется i18n (например, react-i18next) — словари
легко мигрируют.
