# ADR-0002: React Three Fiber для 3D-рендера

Дата: 2025-01-01

## Статус

**Accepted**

## Контекст

Нужно рендерить 3D-карту с экструдированными полигонами стран, освещением, камерой и
интерактивным управлением. Альтернативы:

- **three.js напрямую**: максимум контроля, но много boilerplate для React-приложения
  (imperative ref/DOM-управление, ручной reconcile при смене года);
- **babylon.js**: мощный, но другая экосистема, меньше примеров для React;
- **R3F** (`@react-three/fiber`): декларативный React-рендер поверх three.js, отличная
  интеграция с drei (OrbitControls, Html, etc.).

## Решение

Использовать **@react-three/fiber** + **@react-three/drei**. Декларативное описание сцены
позволяет естественно реагировать на смену года и selectedCountry через React state. drei
даёт готовые `OrbitControls`, `Html` для подписей стран, `Stats` для отладки.

## Последствия

**Плюсы:** идиоматичный React-код, легко добавлять новые объекты (слои, маркеры), активное
сообщество. **Минусы:** дополнительный слой абстракции (отладка R3F сложнее чистого three.js);
рендер идёт через React reconciliation — для тысяч объектов может быть медленнее, но у нас
~30-40 стран, узкое место отсутствует. **Риски:** если понадобится WebGL2-специфичный
функционал вне drei — обход R3F возможен через `<primitive object={...} />`.
